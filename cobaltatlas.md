# Olive Link Catalog

Olive Link Catalog 是一个面向开发者与技术研究人员的轻量级外链资源归集与导航系统。项目本身不存储任何实际内容，而是通过结构化的 Markdown 清单对分散在互联网各处的技术文档、代码仓库、教程笔记和实验性项目进行索引与分类。Olive Link Catalog 适用于需要维护个人或团队技术书签库、构建领域知识图谱、以及基于纯文本文件实现可版本化外链管理的场景。项目定位为“技术外链的元仓库”，帮助用户从信息过载中解脱，快速定位高质量外部资源。

## 功能概览

- 纯文本可版本化索引：所有外链以 Markdown 文件形式组织，天然支持 Git 版本追踪与协作更新。
- 多维度分类标签：每条链接可附带主题标签、难度评级和阅读时长预估，便于按场景筛选。
- 自动健康检查：内置链接有效性检测工具，定期扫描索引中的 URL 并标记失效或变更资源。
- 快速全文检索：基于文件内容实现轻量级 grep 搜索，支持按关键词、文件名或路径定位链接。
- 外部元数据缓存：对每个收录的 URL 自动抓取页面标题与描述，生成摘要缓存以提升浏览体验。
- 导入导出标准格式：支持从 CSV、JSON 和浏览器书签 HTML 文件批量导入，亦可导出为通用格式供其他工具使用。
- 资源变化通知：通过 Webhook 或邮件订阅索引更新，及时获知新增或移除的外链。

## 应用场景

1. 技术团队内部知识库外链管理：团队可将 Olive Link Catalog 部署为内部文档站点的补充模块，集中存放所有外部参考链接、API 文档和第三方工具地址，避免重复收藏与信息散落。

2. 开源项目依赖资源索引：开源维护者可在项目仓库中引用 Olive Link Catalog 来统一管理教程、示例代码和相关生态项目的外链，降低新贡献者的学习门槛。

3. 个人学习路线图构建：学习者可按主题（如 Rust 语言、分布式系统、机器学习）建立独立的索引文件，将零散的博文、视频和官方文档组织成结构化学习路径，并定期回顾更新。

4. 技术资讯聚合与筛选：编辑人员或社区运营者可使用该工具维护每日技术资讯的外链池，经人工筛选后发布为周报或月刊，确保内容质量与相关度。

## 快速开始

以下命令演示了如何获取项目源码、安装依赖并启动本地开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装依赖（Node.js 18+ 环境）
npm install

# 启动本地开发服务器，默认监听 3000 端口
npm run dev
```

访问控制台输出的本地地址即可预览索引界面。若需构建生产版本，请执行 `npm run build` 并将 `dist` 目录部署至 Web 服务器。

## 常见问题

**问：Olive Link Catalog 是否会存储外部资源的内容副本？**  
答：不会。项目仅存储外部资源的 URL 及其元数据（标题、描述、最后访问时间），不保存任何实际文件内容。所有外链内容仍需通过原始 URL 访问，版权归属原发布者。

**问：如何处理索引中的链接失效问题？**  
答：项目内置了定期健康检查机制，可在配置文件中调整扫描频率。当检测到链接返回 4xx 或 5xx 状态码时，系统会在索引文件中标记该链接并发送通知。维护者可手动确认后移除或更新链接。

**问：能否与现有的浏览器书签系统同步？**  
答：可以。项目提供了导入模块，支持解析 Chrome 和 Firefox 导出的 HTML 书签文件，并将其转换为项目内部索引格式。未来版本计划增加双向同步能力。
