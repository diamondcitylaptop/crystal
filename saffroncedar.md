# Olive Resource Atlas

Olive Resource Atlas 是一个面向开发人员与技术研究者的结构化外链与参考资料聚合平台。项目定位于将分散在各类技术社区、代码仓库、个人博客与官方文档中的高质量资源，按照主题域与成熟度进行编目与索引，并提供一致的访问入口。目标用户包括基础软件研究者、编译器与运行时工具链开发者、以及需要频繁查阅底层实现细节的系统架构师。项目本身不托管原始内容，而是通过可追溯的链接与版本标注，建立一份可长期维护的技术资源地图，帮助用户在复杂的技术生态中快速定位到权威、相关且可用的外部信息。

## 功能概览

- 按主题域组织的资源索引：每个资源条目归属一个明确的技术领域，包括语言运行时、垃圾回收算法、编译器中间表示、并发模型、网络协议栈等。

- 基于 Markdown 的目录化存储：所有资源记录以 Markdown 文件形式保存在项目仓库中，支持目录层级浏览与全文检索。

- 资源状态标注与版本追踪：每条外链附带最后检查日期、资源类型（规范、论文、实现笔记、官方文档）及可用性状态。

- 轻量级标签系统：支持为资源添加多个主题标签，便于跨目录过滤与关联查找。

- 简易贡献工作流：通过 Pull Request 提交新增资源或更新已有链接，维护者进行合规性与可用性审核。

- 与 GitHub 原生生态集成：利用 Issues 进行资源失效报告，利用 Actions 实现周期性链接可用性检查。

## 应用场景

场景一：调研新的垃圾回收算法时，需要同时参考多篇学术论文、开源实现代码以及运行时调优指南。Olive Resource Atlas 在 GC 主题目录下汇集了上述三类资源，并标注了对应的实现语言与基准测试环境，可显著缩短前期信息搜集时间。

场景二：团队在进行微服务架构选型时，需要评估多种 RPC 框架的协议设计、序列化性能与多语言支持程度。项目中的网络协议与框架目录按照语言绑定丰富度与维护活跃度对资源进行了分组，支持快速横向对比。

场景三：编译器课程的教学准备过程中，讲师需要为学生提供一份涵盖词法分析、语法分析、中间代码生成与目标代码生成的课外阅读清单。项目中的教育索引目录收录了多所高校的公开课件与实验指导链接，并附有难度分级建议。

## 快速开始

以下命令可在本地完整克隆项目仓库并启动静态索引页面服务。

```bash
git clone https://github.com/zerxonhy/olive.git
cd olive/resource-atlas
npm install
npm run build
npm run serve
```

执行上述命令后，本地服务默认监听 3000 端口，可通过浏览器访问 http://localhost:3000 查看资源索引主页。若只需使用命令行检索，可直接在项目根目录下运行 `./bin/search.js --keyword <关键词>`。

## 贡献指南

1. 从项目仓库复刻（Fork）主仓库至个人账户，并在本地克隆复刻版本。建议在新建的分支上进行修改，分支命名格式为 `feature/resource-add-<资源简称>` 或 `fix/update-link-<资源简称>`。

2. 在 `resource-atlas/data/entries/` 目录下新增或修改对应的 JSON 元数据文件。新增条目需包含必填字段：资源标题、原始 URL、主题域、标签列表、添加日期和资源类型。修改已有条目时，需在元数据中更新 `last_verified` 字段并注明变更原因。

3. 本地运行 `npm run test` 和 `npm run check-links` 确保新增或修改的资源链接可访问，且元数据结构符合 schema 定义。若测试失败，请根据错误提示调整条目内容。

4. 提交变更并推送到个人复刻仓库，随后在 GitHub 上向主仓库发起 Pull Request。PR 描述中需清晰说明新增资源的价值或原有链接的变更原因，并引用相关的 Issue 编号（如有）。

5. 等待维护者审核。审核通过后，PR 将被合并，新增资源会在下一个构建周期自动出现在索引页面中。若审核未通过，维护者会在 PR 中留言说明修改要求，请根据反馈进行调整后再次提交。

## 常见问题

问：项目中的资源链接失效了，应该怎么办？

答：请先在 GitHub Issues 中使用 “broken link” 标签提交一个问题，并附上失效链接所在的条目名称与完整 URL。维护者会定期处理失效报告，或在您提交 PR 修复后合并变更。若您希望自行修复，可参照贡献指南修改对应条目的 `url` 字段并将 `status` 更新为 `broken`。

问：如何请求新增一类主题域的资源索引？

答：请先在 Issues 中提出主题域新增建议，说明该领域的核心范围、已知权威资源来源以及目标用户群体。维护者会评估该主题域是否与项目现有定位重叠，并讨论是否在目录结构中新增顶层分类。在讨论达成一致后，您可以按照贡献指南提交首批资源条目。

问：本地构建索引页面时出现 Node.js 内存不足错误，如何解决？

答：当资源条目数量增长至数千条时，构建脚本可能消耗较多内存。您可以在执行 `npm run build` 前设置环境变量 `NODE_OPTIONS=--max-old-space-size=4096` 以增加内存限制。若仍然出现错误，请检查是否有重复条目导致构建循环引用，并可尝试使用 `--no-cache` 选项禁用缓存重新构建。
