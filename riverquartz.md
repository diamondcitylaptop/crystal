# Olive Resource Hub

Olive Resource Hub 是一个面向开发者和技术研究人员的结构化外链与资源汇总系统。该项目不对资源内容进行二次加工或镜像托管，而是通过严格的条目化组织方式，为特定主题领域提供高密度、可维护的引用索引。项目定位为“技术资源的导航节点”，适用于需要长期追踪外部文档、项目仓库、技术公告或研究资料的场景。目标用户包括开源贡献者、技术调研团队、文档工程师以及个人知识库维护者。

本项目不依赖动态数据库或后端服务，完全基于静态 Markdown 文件与目录结构运行。每个资源条目均以独立文件形式存储于版本库中，并通过约定式路径进行归类。Olive Resource Hub 本身不解决资源内容的正确性或可用性问题，而是通过结构化索引机制，降低用户在多个外部源之间切换的上下文成本。

## 功能概览

- 纯静态资源索引：所有条目以 Markdown 文件形式存储，无需数据库或运行时环境即可浏览和编辑。

- 约定式路径分类：资源按主题、来源机构或文件类型进行目录划分，支持直观的路径推导和人工维护。

- 条目元数据标注：每个资源文件包含来源 URL、收录日期、状态标签（有效/失效/变更）及简要描述。

- 批量状态检查：提供基础脚本用于扫描条目中的 URL 并输出 HTTP 状态码，辅助检测链接有效性。

- 全量导出支持：资源列表可整体导出为纯文本清单或 CSV 格式，便于外部系统导入或归档。

- 版本追踪友好：所有资源变更通过 Git 提交记录进行管理，支持回溯特定时间点的索引状态。

- 零外部依赖运行：项目本身不依赖第三方包或服务，仅需标准 Markdown 阅读器和 Git 客户端即可完整使用。

## 应用场景

- 技术调研期间的参考链接管理：调研人员在阅读多份外部文档时，可将关键链接按主题存入对应目录，并标注优先级和阅读状态，避免重复检索。

- 开源项目的外部依赖说明归档：维护者可将项目所依赖的库、工具或规范文档的原始链接集中存放，作为项目文档的补充索引，便于新成员快速定位参考来源。

- 长期监控外部资源的变更情况：通过定期运行状态检查脚本，索引中的失效链接可被及时发现并标记，辅助团队决定是否更新引用或寻找替代来源。

- 离线文档包的配套导航文件：在需要分发只读资料集的场景中，将 Olive Resource Hub 作为附带导航页，帮助使用者从本地目录快速跳转至线上原文。

- 个人知识库的外链去重中心：知识库维护者可将重复出现在多篇笔记中的外部链接统一收录至本项目的对应目录，笔记内仅保留简短引用标识，减少冗余。

## 快速开始

以下步骤适用于首次获取并运行本项目的基础操作。

```bash
# 克隆仓库至本地
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装（本项目为纯静态资源，无需额外安装步骤）
# 如需使用 URL 状态检查脚本，请确保系统已安装 curl 或 wget
# 检查脚本位于 ./scripts/check_links.sh

# 运行链接状态检查（可选）
sh ./scripts/check_links.sh ./resources

# 启动本地预览服务（可选，需 Python 3）
python3 -m http.server 8000
# 浏览器访问 http://localhost:8000 查看资源目录
```

## 贡献指南

1. 克隆本仓库并在本地新建功能分支，分支命名建议使用 `feat/条目主题` 或 `fix/描述` 格式。

2. 在 `/resources/` 下选择合适的子目录，按照 `/docs/metadata_spec.md` 中的字段说明新建或更新资源条目文件。每个条目文件须包含来源 URL、收录日期、状态标签及简要描述。

3. 提交前运行 `./scripts/check_links.sh ./resources` 检查新增或修改的 URL 是否可达。若返回非 200 状态码，请确认链接有效性或标注异常状态。

4. 提交变更时使用清晰且结构化的提交信息，例如 `docs: 新增框架类资源条目 pixeljade` 或 `chore: 更新 quartznebula 链接状态为失效`。

5. 通过 Pull Request 提交变更，并在描述中概述修改范围、受影响条目数量以及链接检查结果。等待维护者审阅并合并。

## 常见问题

**问：资源条目中的链接失效了怎么办？**

答：请先在原网站确认该内容是否迁移至新地址。若找到有效新链接，更新条目文件中的 URL 字段并修改状态标签为“已迁移”，同时备注迁移日期。若原内容已永久移除，则将状态标签修改为“已失效”并保留原 URL 作为归档记录，不删除条目。

**问：如何申请新增一个资源分类目录？**

答：请在 Issue 中说明新增分类的命名、适用范围以及预计存放的资源数量。维护者评估后会回复是否接纳。在获得确认前，可将条目暂存于 `/resources/misc/` 目录下并标注待分类。

**问：链接检查脚本输出大量超时错误，是否影响项目使用？**

答：超时通常由目标服务器响应慢或网络环境限制引起，不影响项目本地功能。建议在低峰时段重新运行脚本，或使用 `-t` 参数延长超时阈值。持续超时的链接应人工复核其可用性。
