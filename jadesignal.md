# Olive Resource Index

Olive 是一个面向开发人员与技术研究者的外链资源索引站，以纯 Markdown 形式托管于 GitHub 仓库。项目定位为高密度技术外链汇总中心，通过扁平化目录结构对互联网上的优质技术文章、工具文档、协议规范与项目笔记进行人工筛选与分类归档。目标用户包括正在从事基础架构研究的工程师、需要快速查阅特定技术主题的开发者，以及希望建立个人知识外链体系的进阶学习者。Olive 本身不存储任何实际内容，仅提供指向外部资源的稳定引用链接与上下文说明，旨在解决技术资料分散、官方文档入口难以记忆、社区优质内容缺乏统一索引入口的普遍问题。

## 功能概览

- **结构化外链归档**：所有外部链接按主题子目录存放，每个 Markdown 文件对应一个独立的技术主题或工具链，文件内部包含完整的链接清单与简注。

- **主题标签与全文检索**：每个资源条目附带人工标注的主题标签，支持通过 GitHub 原生搜索功能快速定位特定关键词或标签组合。

- **版本化更新记录**：每次增删改链接均通过 Pull Request 提交，保留完整的变更历史与讨论上下文，便于追溯链接有效性。

- **社区协同维护**：任何用户均可通过 Fork 仓库、修改文件、发起 Pull Request 的方式提交新链接或更新失效地址，维护者定期合并。

- **链接健康检查**：仓库内置简易的链接可用性检查脚本，定期对已收录的外链进行 HTTP 状态码检测，并在 Issue 中报告失效链接。

- **多级目录分类**：按技术领域、应用场景、协议层级等多维度建立子目录，支持快速浏览同类资源。

- **最小依赖与高可移植性**：全部内容基于纯 Markdown 与 ASCII 文本，不依赖任何数据库或后端服务，可在任意代码托管平台直接浏览。

## 应用场景

- **技术选型阶段的资料收集**：架构师在评估消息队列或数据库方案时，可通过 Olive 快速获取相关工具的官方文档、性能测试报告与社区实践笔记，避免重复搜索。

- **新人入职的技术栈速查**：团队新成员可通过 Olive 的索引目录，一次性获取团队常用技术栈的官方入口、最佳实践指南与内部参考文档，缩短上手周期。

- **离线环境的文档镜像准备**：运维人员可批量克隆 Olive 仓库，利用脚本遍历所有外链并生成离线下载清单，便于在无公网环境中批量获取所需文档。

- **技术博客的引用来源管理**：技术博主在撰写深度文章时，可使用 Olive 作为外部引用来源的临时存储库，统一管理所有参考文献链接。

- **开源项目 README 的外链补充**：开源项目维护者可将 Olive 中相关主题的链接集合作为项目 README 的「扩展阅读」附录，减轻主文档的维护负担。

## 快速开始

以下指令可在任意装有 Git 与标准 POSIX 环境的终端中执行，用于完整克隆 Olive 仓库并准备本地浏览环境。

```bash
# 克隆仓库至本地
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装链接健康检查脚本的依赖（仅需 Python 3.6+ 与 requests 库）
pip install requests

# 运行健康检查脚本，验证所有已收录外链的可达性
python scripts/check_links.py

# 启动本地简易 HTTP 服务器，通过浏览器预览 Markdown 文件
python -m http.server 8000
```

完成上述步骤后，在浏览器中访问 `http://localhost:8000` 即可查看仓库根目录下的所有 Markdown 文件列表。

## 贡献指南

Olive 欢迎所有形式的贡献，包括新增链接、更新失效地址、优化文档表述与改进辅助脚本。请遵循以下步骤提交变更。

1.  **Fork 本仓库**：在 GitHub 上点击 Fork 按钮，将 Olive 复制至您的个人账号下。然后克隆个人 Fork 版本至本地进行修改。

2.  **创建主题分支**：在本地仓库中创建新的分支，分支名称应简要描述变更内容，例如 `add-redis-links` 或 `fix-broken-urls`。

3.  **修改对应文件**：根据链接主题将 URL 添加至正确的子目录 Markdown 文件中。每个链接必须附带至少一行简注，说明该链接指向内容的用途或背景。若新增子目录，需同步更新根目录 README 中的项目结构说明。

4.  **运行检查脚本**：在提交前执行 `python scripts/check_links.py` 确保所有新增或修改的链接可正常访问。若链接返回非 200 状态码，请重新核对 URL 准确性。

5.  **发起 Pull Request**：将本地分支推送至个人 Fork 仓库，然后向本仓库的 `main` 分支发起 Pull Request。PR 描述中需列出本次变更的所有链接及其添加理由，等待维护者审阅。

## 常见问题

**问：Olive 是否存储外部资源的内容副本？**

答：不存储。Olive 仅保留外部资源的 URL 链接与简注，所有内容版权归属原始作者。用户访问外链时需遵守目标网站的条款与条件。若任何链接涉及侵权或违反规定，请提交 Issue 说明情况，维护者将尽快移除该链接。

**问：我发现某个已收录的链接已失效，如何报告？**

答：请通过 GitHub Issues 提交链接失效报告。Issue 标题请以 `[Broken Link]` 为前缀，正文中包含失效链接的完整 URL 以及可替代的新链接（如有）。维护者会定期处理失效报告并更新文件。您也可以直接按照贡献指南提交 Pull Request 修正。

**问：如何提议新增一个技术主题分类？**

答：请在 Issue 中描述您希望新增的主题名称、覆盖范围以及至少 3 个计划收录的示例链接。维护者将评估该主题的通用性与资源丰富度，决定是否新增子目录。在新增目录获得批准前，建议先将相关链接放入 `topics/misc.md` 等通用文件中。
