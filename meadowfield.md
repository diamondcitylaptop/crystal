# Olive Resource Index

Olive Resource Index 是一个面向开发者与技术研究人员的结构化外链与技术资源汇总工具。项目本身不存储任何实际内容，仅提供高度组织化的导航索引与元数据描述，协助用户快速定位分布在多个仓库与文档站点中的技术手册、配置模板、协议参考实现与架构说明。项目定位为技术资源的“元目录”，适用于需要频繁查阅分散资料的运维工程师、架构师与开源贡献者。

## 功能概览

- 多源资源聚合：支持将不同 GitHub 仓库中的 Markdown 说明文档统一编入索引，形成单一查阅入口。
- 主题标签分类：每个资源条目附带主题标签，覆盖网络协议、存储后端、消息队列、可观测性、安全基线等领域。
- 原始链接直出：所有外链保留原始 URL 格式，不添加重定向、短链或中间跳转页，确保访问路径透明可溯。
- 版本锚点提示：对引用外部文档的链接标注参考提交哈希或日期范围，便于追溯内容变更。
- 本地镜像缓存建议：提供可选的本地缓存策略说明，帮助用户在内网环境部署只读副本。
- 周期性校验机制：内置链接可用性校验脚本示例，支持定时检测失效资源并输出报告。
- 轻量级元数据扩展：允许用户为每条记录附加自定义字段，如优先级、所属团队或合规等级。

## 应用场景

1. 内部技术文档门户的补充索引：企业技术团队可将 Olive Resource Index 作为统一入口，集中管理散落在多个团队 Wiki 或代码库中的关键设计文档，减少查找时间。

2. 开源项目依赖链分析：开源维护者通过索引中整理的协议参考与实现说明，快速追溯上游依赖的技术细节，辅助版本升级与兼容性评估。

3. 新人入职学习路径规划：团队新人可根据索引中按主题聚合的资源列表，按顺序阅读网络基础、存储原理、可观测性实践等材料，形成系统化认知。

4. 离线环境资源预加载：在受限网络环境中，运维人员可依据索引清单提前下载所需文档，打包放入内部镜像站，保障生产变更前的资料查阅需求。

## 快速开始

以下步骤帮助您在本地启动 Olive Resource Index 的静态实例，用于预览或二次开发。

```bash
# 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装依赖（基于 Node.js 22 LTS）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

## 贡献指南

1. 复刻项目仓库至个人账号，并在本地创建功能分支，分支命名建议采用 `feat/功能描述` 或 `fix/问题描述` 格式。

2. 新增资源条目时，请依照现有模板在对应的 Markdown 文件中补充标题、标签与描述字段，并确保所有外链以原始格式写入，不添加任何协议或域名改写。

3. 提交代码前运行 `npm run lint` 与 `npm run test` 确保代码风格一致且所有测试用例通过。

4. 发起 Pull Request 至主仓库的 `main` 分支，并在 PR 描述中清晰说明新增或修改的资源列表，以及所影响的场景。

5. 项目维护者将在 3 个工作日内进行审核，如有修改意见将通过 PR 评论区反馈。

## 常见问题

**问：索引中的资源链接失效怎么办？**

答：您可以使用项目根目录下 `scripts/validate-links.sh` 脚本对当前所有收录链接进行批量可用性检测。若发现失效链接，请通过 Issues 页面报告，或按照贡献指南提交 PR 更新对应条目。

**问：能否在完全离线环境中使用该索引？**

答：可以。首次完成依赖安装与构建后，所有前端资源均可在本地运行。但索引中收录的外部链接本身仍需要网络访问才能获取实际内容。您可结合内部镜像或本地缓存策略解决内网访问问题。

**问：如何自定义索引的主题标签体系？**

答：标签定义位于 `src/core/parser.js` 中的 `TAG_DICTIONARY` 常量对象。您可以按需增加或修改标签键值对，并确保所有资源条目标注的标签均存在于该字典中，否则会被归入“其他”类别。
