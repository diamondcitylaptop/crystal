# Olive Index

Olive Index 是一个面向技术研究与开发人员的外链资源整理与导航系统。本项目不直接存储任何原始文档或二进制内容，而是以结构化索引的方式，对分散在各类代码托管平台、技术博客与文档站点中的优质资源进行归类、标注与关联，帮助用户从大量低信噪比的网络信息中快速定位到高价值技术资料。Olive Index 定位为技术决策辅助工具，适用于需要频繁查阅底层实现细节、版本差异对比或协议行为分析的工程师、架构师及技术管理者。

本项目采用纯静态索引机制，所有资源条目均以 Markdown 文件形式维护于仓库目录中，支持通过 Git 进行版本追踪与协同编辑。Olive Index 不依赖外部数据库或运行时服务，克隆即用，内容可离线完整查阅，适合集成至企业内部文档体系或个人知识管理工作流。

## 功能概览

**结构化资源索引**：以目录树和多级标签体系组织外部技术文档链接，每个条目包含原始 URL、概要描述、关键词标签及采集时间。

**原始内容透传引用**：所有外部链接保持原始 URL 格式输出，不进行重定向、短链压缩或中间页跳转，确保引用路径的确定性与可追溯性。

**多维度筛选视图**：基于文件名后缀、目录前缀及关键词匹配提供若干预定义视图，例如协议分析视图、组件版本追踪视图和安全公告聚合视图。

**离线可用的静态数据**：全部索引数据以纯文本 Markdown 格式存储，无需网络或后台服务即可进行全文检索与浏览。

**版本化变更记录**：每个资源条目的新增、修改或废止均通过 Git 提交记录关联，便于审计资源演进历史。

**零运行时依赖**：本项目仅为数据组织仓库，不包含任何后端程序、数据库或前端构建工具，降低维护成本与安全风险。

## 应用场景

技术方案选型评估期间，团队需要横向对比多个中间件或库的版本特性。通过 Olive Index 中的版本追踪链接，可直接跳转至对应项目的具体发布页面或变更清单，减少在项目主页与发行版页面之间的反复跳转时间。

安全事件应急响应过程中，工程师需要迅速查阅某个依赖组件的官方安全公告或补丁提交记录。Olive Index 维护了与安全相关的参考链接子集，支持按关键词快速过滤，加速根因分析。

架构评审或技术分享准备阶段，讲者需要收集多个外部参考资料作为论据支撑。利用 Olive Index 的索引条目，可一次性导出相关主题的链接清单，并按照目录结构组织为引用附录。

离线环境或内网开发场景下，开发人员无法依赖在线搜索引擎。将 Olive Index 仓库完整镜像至内部 Git 服务后，所有索引数据可离线浏览，确保信息获取不受网络条件限制。

## 快速开始

以下命令序列可在任意 POSIX 兼容环境中完成本项目的克隆与初始配置。由于本项目仅为静态索引数据，无需额外安装步骤或构建过程。

```bash
git clone https://github.com/zerxonhy/olive.git
cd olive
mkdir -p .olive/views
cp -r examples/ .olive/views/ 2>/dev/null || true
echo "Olive Index 已就绪，当前索引条目位于根目录 Markdown 文件中。"
```

如需验证索引文件完整性，可运行内置校验脚本（若有）。否则直接通过文本编辑器或 `grep` 命令浏览内容。

```bash
grep -c "^https://" *.md || echo "当前目录下无 Markdown 文件"
```

## 贡献指南

Olive Index 欢迎外部贡献者新增或更新资源链接。所有贡献需遵循以下步骤以确保索引质量与一致性。

第一，从本仓库复刻一份个人副本，并在本地克隆该复刻版本。创建新的特性分支，分支名称需与本次变更主题相关，例如 `add-delta-gamma` 或 `update-ember-urls`。

第二，在根目录下创建或修改对应的 Markdown 索引文件。每个文件须以标题行开头，说明该文件覆盖的主题领域。文件内部的链接列表每行仅放置一个 URL，且必须保持原始格式。每个链接上方或下方应附带一行简短的描述性注释，格式为 `<!-- 描述：xxx -->`。

第三，运行本地校验脚本（若存在）检查链接格式与描述完整性。若没有校验脚本，需手动确认所有新增链接均可访问，且内容主题与文件标题领域一致。

第四，提交变更并推送到个人复刻仓库，随后通过 GitHub 界面发起拉取请求。拉取请求标题应简明扼要描述变更范围，正文中需列出本次新增或修改的具体链接数量及影响范围。

第五，等待维护者审核。审核通过后，变更将被合并至主分支，并随下一次版本标签发布生效。若审核未通过，维护者会在拉取请求中标注修改意见，贡献者需根据反馈调整后重新提交。

## 常见问题

**问：Olive Index 是否存储外部链接对应的完整内容副本？**

答：不存储。本项目仅维护外部资源的原始 URL 及其描述性元数据，不托管任何第三方文档、代码或二进制文件的副本。用户访问链接时，实际内容由原始站点提供，Olive Index 不承担内容可用性或正确性的保证责任。若某个链接失效，欢迎通过贡献流程更新或移除该条目。

**问：为什么某些链接使用的是裸域名或非标准协议？**

答：Olive Index 严格保留贡献者提交时的原始 URL 形式，不进行任何规范化改写。这一策略旨在防止自动补全或协议升级导致的目标地址非预期变化。用户在使用时，如果发现链接无法访问，可先尝试手动补充协议头或调整大小写，但索引条目本身将以原始字符串形式继续保留，除非经贡献流程明确修正。

**问：如何批量导入大量外部链接？**

答：当前版本不提供自动导入工具。对于批量新增需求，建议按照贡献指南中的模板，逐条编写索引文件。若数量极大，可先在本地编写脚本生成符合格式的 Markdown 片段，再手动审查后合并。维护者欢迎此类批量贡献，但要求每个批次不超过 20 条，以便逐项复核链接质量。
