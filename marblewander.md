# Olive Collective

Olive Collective 是一个面向开发者、技术研究者和开源爱好者的技术资源外链汇总与导航项目。该项目不直接存储任何大型二进制文件或版权内容，而是通过维护一套精心筛选的 Markdown 文档索引，将分散于网络各处的优质技术教程、工具文档、架构分析笔记和社区讨论串聚合于统一的目录体系之下。项目定位为“技术阅读路径的起点”，帮助用户从单一入口快速触达特定技术栈的深度资源，避免在搜索引擎和社交信息流中反复过滤噪声。目标用户包括正在学习系统设计的中高级工程师、需要快速查阅特定框架边缘案例的维护者，以及希望建立系统性知识图谱的技术作者。

## 功能概览

- 分层索引导航：按技术领域、难度等级和资源形态（教程/案例/参考手册）对链接进行三级标签化组织，支持快速过滤。
- 锚点摘要生成：为每个收录的外部链接自动提取或人工编写一段上下文摘要，说明该资源解决什么问题、适合什么阶段阅读。
- 版本关联追踪：对于涉及框架、语言版本特性的资源，标注其适用的最低版本或弃用警告，避免用户参考过时实践。
- 阅读路径推荐：基于常见学习路径（如“从零搭建微服务”、“并发模式入门”）将多个链接串联为有序列表，降低碎片化阅读成本。
- 社区反馈集成：每个链接条目下可嵌入来自 GitHub Issues 的讨论摘要，展示其他开发者对该资源的实际使用感受或勘误记录。
- 周期健康检查：通过自动化工作流定期检测链接可用性及内容变更，标记失效或大幅改动的资源，并在文档头部生成状态徽章。

## 应用场景

- 技术选型调研：团队在确定微服务框架或数据库中间件之前，可通过 Olive Collective 的“选型对照”阅读路径，一次性浏览多个对比分析帖、官方性能测试报告和社区迁移案例，大幅压缩资料收集时间。
- 架构复习备考：计划参加系统设计面试或架构师认证的工程师，可以利用项目中的“高并发设计模式”索引，系统性地阅读分布式缓存、消息队列、限流降级等方向的经典文章，配合每篇摘要中的思考题进行自查。
- 文档离线准备：在进行网络受限环境的开发或演示前，用户可以借助项目的链接汇总快速确定必须下载的文档包、镜像地址或配置模板，通过脚本批量获取后放入本地知识库。
- 开源贡献入门：新手贡献者通过项目中的“如何阅读源码”路径，找到目标项目的贡献指南、核心模块导读和典型 PR 拆解，降低理解庞大代码基底的认知负担。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/your-org/olive-collective.git

# 进入项目目录
cd olive-collective

# 安装依赖（主要用于本地预览和链接检查）
npm install --global markdown-link-check

# 运行本地预览服务（如已配置静态站点生成器）
# 若使用 VitePress / VuePress 等工具，请参考对应命令
npm run docs:dev
```

## 项目结构

```
├── config/                          # 站点配置文件（如 VitePress 配置）
├── docs/                            # 项目自身文档（非索引内容）

```

## 贡献指南

1. 在 `indexes/by-domain/` 下找到对应技术领域的 Markdown 文件，若不存在则新建，遵循现有文件头部的元数据格式（包含 `domain`、`level`、`type` 三个字段）。
2. 在文件末尾按照模板添加新资源条目，必须包含外部链接原始 URL、一段不超过 200 字的摘要说明以及资源发布年份。
3. 若资源涉及多个领域或路径，请在 `paths/` 目录下对应路径文件中同步添加该资源的引用并说明其在路径中的顺序与前置条件。
4. 提交 Pull Request 前，运行本地链接检查脚本 `npm run check-links` 确保所有已有链接均有效；若新增链接失效，需替换或修复后再提交。
5. PR 描述中请注明本次添加资源所属批次、是否关联某个 Issue 以及是否有已知的访问限制（如需要科学上网或登录）。

## 常见问题

Q: 项目中的链接失效了怎么办？

A: 请直接在 GitHub Issues 中使用“链接失效报告”模板提交，我们每周例行检查中也会自动标记失效链接。在修复之前，失效链接条目将被打上 `[deprecated]` 前缀且不会出现在推荐阅读路径中。您也可以自行提交 PR 替换为有效的替代链接。

Q: 我可以推荐自己的博客文章或项目文档吗？

A: 可以，但需要满足两条基本准则：第一，内容必须与现有索引领域高度相关且不重复已有资源；第二，摘要必须客观描述文章解决的问题，不得使用过度营销措辞。审核时维护者会根据质量与相关性决定是否收录。

Q: 项目是否接受非英文资源？

A: 接受。非英文资源需在摘要中标注语种，并且要求摘要本身使用英文或中文撰写以确保索引可读性。对于中文资源，当前收录比例约为 30%，后续将逐步扩展多语言覆盖。
