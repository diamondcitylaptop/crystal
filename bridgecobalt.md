# Olive Index

Olive Index 是一个面向技术研究、安全审计与开源情报（OSINT）场景的轻量级外链资源汇总工具。项目本身不存储任何实质性内容，仅提供结构化索引，将分散于 GitHub 仓库、技术社区与自建文档站点中的高价值外部链接进行归集与分类。项目定位为个人研究者与小型团队使用的导航层，通过 Markdown 文件维护索引规则，支持静态站点生成器或纯脚本解析两种使用方式。目标用户包括安全研究员、渗透测试工程师、CTF 选手以及需要快速检索外部技术资料的开源贡献者。

项目解决的核心问题是信息分散与链接失效。在长期技术研究过程中，大量有价值的参考链接散落在临时笔记、浏览器书签或聊天记录中，缺乏统一的版本管理与分类逻辑。Olive Index 将所有外链以纯文本形式托管于 Git 仓库，利用提交历史追踪链接变更，通过脚本自动化检查链接可达性，并提供多维度标签过滤能力。项目本身不依赖数据库或外部服务，完全基于文件系统运行，确保在离线环境下仍可全文检索已索引的链接描述信息。

## 功能概览

- **多层级索引目录** 支持按技术领域、风险等级、数据来源等维度建立三级嵌套分类，每个分类对应独立的 Markdown 索引文件，便于维护与审核。

- **链接状态自动检查** 内置 Python 脚本定时检测索引中所有外链的 HTTP 状态码，自动标记失效链接并生成报告，协助管理员及时更新或移除无效资源。

- **标签与全文检索** 每个索引条目支持自由标签附加，配合简单的 grep 或 ripgrep 命令即可实现毫秒级全文搜索，无需启动任何后台服务。

- **静态站点生成适配** 索引文件遵循约定的 frontmatter 格式，可无缝对接 Hugo、VuePress 或 MkDocs 等静态站点生成器，一键生成带搜索功能的在线导航页面。

- **版本化变更追踪** 所有索引更新通过 Git 提交记录完整保留，支持回滚至任意历史版本，方便审计某一时间点的外部资源引用情况。

- **自定义元数据扩展** 每条索引支持扩展字段，包括但不限于维护责任人、最后验证时间、重要性等级、相关 CVE 编号或技术标准编号，满足企业级内部知识库管理需求。

- **命令行交互界面** 提供交互式 CLI 工具，支持模糊搜索、按标签过滤、批量添加新链接以及导出为 JSON/CSV 格式，便于与其他分析工具集成。

## 应用场景

- **安全漏洞研究资料归档** 当分析某一 CVE 编号的漏洞时，研究者可以将相关的 PoC 仓库地址、厂商公告链接、第三方分析文章以及补丁差异对比页面全部归集到同一索引条目下，通过标签标记漏洞类型与影响版本。后续复现或撰写报告时可直接从索引中批量打开所有参考资料，避免重复检索。

- **CTF 竞赛知识库构建** 参赛团队可将历年比赛中的 Writeup 链接、常用工具下载地址、在线解密服务页面以及特定语言的反序列化 payload 仓库统一索引。在实战中通过 CLI 工具快速搜索关键字，秒级定位所需资源，大幅提升解题效率。

- **企业内部技术文档导航** 中型企业的架构团队可使用 Olive Index 维护微服务治理、日志规范、监控面板地址、运维操作手册等内部资源的入口索引。配合静态站点生成器部署为团队内部导航页，新入职员工可快速了解技术栈全貌，减少重复提问。

- **开源项目依赖追踪** 项目维护者可将所有上游依赖的官方文档、issue 跟踪器、CI 状态页面以及安全公告 RSS 链接纳入索引。定期运行链接检查脚本可及时发现文档迁移或域名过期情况，提前规避构建流程中断风险。

- **学术文献与数据源管理** 科研人员在撰写综述或进行数据分析时，可将数据集下载地址、统计年鉴官网、相关论文的开放获取链接以及代码实现仓库集中索引。每条记录附加研究主题标签与引用格式备注，方便协作成员统一数据来源。

## 快速开始

以下命令演示如何在本地环境获取 Olive Index 源码、安装基础依赖并启动示例索引服务。

```bash
# 克隆项目仓库
git clone https://github.com/zerxonhy/olive.git
cd olive

# 安装 Python 依赖（链接检查与 CLI 工具）
pip install -r requirements.txt

# 运行本地索引预览（生成静态站点到 public/ 目录）
python scripts/build.py --source ./index --output ./public --serve
```

执行完成后，访问命令行输出的本地地址（默认为 http://127.0.0.1:5500）即可查看示例索引页面。若只需使用 CLI 检索功能，可直接运行 `python scripts/search.py "关键字"` 进行快速查询。

## 贡献指南

1. 复刻本项目仓库至个人账号，在本地克隆复刻后的副本。创建新的功能分支，分支命名遵循 `feature/简述变更内容` 或 `fix/简述修复问题` 格式，确保与主分支保持清晰隔离。

2. 新增索引条目时，请严格参照 `docs/index-spec.md` 中的字段定义，确保 YAML frontmatter 完整无误。若涉及新增分类目录，需同步更新 `index/` 下的目录结构，并在对应的父级索引文件中添加引用关系。

3. 提交变更前运行测试套件 `python -m pytest tests/`，确保所有单元测试通过。同时执行链接检查脚本 `python scripts/check_links.py --full` 验证新增或修改的外链均返回有效的 HTTP 状态码（2xx 或 3xx）。

4. 提交信息采用约定式提交规范，格式为 `<type>(<scope>): <subject>`，其中 type 包括 feat、fix、docs、chore 等，scope 指明影响的索引分类或模块。提交后推送至个人复刻仓库，然后通过 GitHub 界面发起 Pull Request 至主仓库的 main 分支。

5. 项目维护者将在 3 个工作日内评审 Pull Request，评审意见可能涉及索引条目分类调整、描述补充或链接替换。根据反馈修改后再次推送，直至评审通过并合并。合并后贡献者名称将被记录于 `CONTRIBUTORS.md` 文件中。

## 常见问题

**问：索引中的链接访问时返回 403 或 404 状态码，项目是否会自动处理？**

答：链接检查脚本仅负责检测状态码并生成报告，不会自动删除或修改索引条目。管理员应根据报告手动核实链接有效性：对于临时故障的链接可等待恢复；对于永久迁移的链接需更新为目标地址；对于彻底失效的链接应从索引中移除，并在提交信息中注明原因。建议配置每周定时任务运行检查脚本，保持索引质量。

**问：项目是否支持多语言索引条目描述？**

答：索引规范中的 description 字段为纯文本字符串，未限制编码格式，因此完全支持 UTF-8 编码的任何语言。建议团队内部统一采用英文或中文撰写，避免混用造成检索时关键字遗漏。若需要多语言版本，可创建不同分支或在 frontmatter 中扩展 multilingual 字段，但官方构建流程默认仅编译默认语言描述。

**问：静态站点生成后，搜索功能无法找到部分索引条目，可能原因是什么？**

答：首先确认索引文件位于 `index/` 目录下且扩展名为 `.md`，构建脚本默认仅扫描该目录的一级子文件夹。其次检查 frontmatter 中的 title 和 tags 字段是否包含期望的搜索关键字，部分生成器仅索引显式声明的字段。若使用 Hugo 适配层，请确认 `config.toml` 中已启用 `disableKinds` 未排除 taxonomy 页面。最后运行 `python scripts/search.py --rebuild-index` 强制重建本地搜索缓存后重试。
